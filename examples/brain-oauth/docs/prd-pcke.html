🔐 Brain OAuth · PKCE 接入文档

为纯前端应用（SPA、移动端、桌面应用）提供安全的 OAuth 2.0 授权码流程，无需 client_secret，支持 PKCE 防拦截攻击。

📖 概述

Brain OAuth 现在支持 PKCE (Proof Key for Code Exchange) 扩展（RFC 7636），允许公共客户端（如浏览器 SPA、Electron、React Native）在不暴露 client_secret 的情况下完成安全的授权码流程。你只需注册一个公共客户端，即可在前端直接集成。

✅ PKCE 流程完全由前端 JavaScript 完成，无需后端服务器。Brain OAuth 服务端会验证 code_challenge 和 code_verifier，防止授权码被拦截后恶意使用。

🔑 前置条件

  1. 访问 Brain OAuth 开发者控制台，使用 Brain 账号登录。
  2. 点击「创建新应用」，填写应用名称、重定向 URI（例如 https://yourapp.com/callback 或 http://localhost:3000/callback）。
  3. 在创建或编辑应用时，将「客户端类型」选择为「公共客户端」（不生成 client_secret）。系统会分配一个 client_id，无需保管 secret。
  4. 记录下你的 client_id 和注册的 redirect_uri。
⚠️ 如果应用类型误选为「机密客户端」，PKCE 流程仍可工作，但你需要提供 client_secret（不安全）。请务必选择「公共客户端」。

🧩 PKCE 流程原理

PKCE 在标准授权码流程上增加了动态密钥验证：

  • code_verifier：前端生成的高熵随机字符串（43~128 字符）。
  • code_challenge：对 verifier 进行 SHA256 哈希并 base64url 编码后的值。
  • 授权请求携带 code_challenge 和 code_challenge_method=S256。
  • 授权码返回后，前端用 code_verifier 换取 token，服务端验证哈希匹配后颁发 token。

即使授权码被窃取，攻击者没有 code_verifier 也无法换取令牌。

📦 快速接入代码（原生 JavaScript）

以下是一个完整的 HTML 示例，演示 PKCE 完整流程：生成挑战码 → 跳转授权 → 回调处理 → 换取 token → 调用 /userinfo。你只需修改 CLIENT_ID 和 REDIRECT_URI。 

<!DOCTYPE html>
<html>
<head>
    <title>Brain OAuth PKCE 演示</title>
    <meta charset="UTF-8">
</head>
<body>
    <h1>Brain OAuth 登录 (PKCE)</h1>
    <button id="loginBtn">使用 Brain 账号登录</button>
    <pre id="result"></pre>

    <script>
        // ========= 配置 =========
        const CLIENT_ID = '你的公共客户端client_id';
        const REDIRECT_URI = 'http://localhost:3000/callback'; // 必须与注册时一致
        const AUTHORIZE_URL = 'https://your-brain-oauth.com/oauth/authorize';
        const TOKEN_URL = 'https://your-brain-oauth.com/oauth/token';
        const USERINFO_URL = 'https://your-brain-oauth.com/userinfo';

        // ========= PKCE 工具函数 =========
        function generateRandomString(length) {
            const possible = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789-._~';
            let text = '';
            for (let i = 0; i < length; i++) {
                text += possible.charAt(Math.floor(Math.random() * possible.length));
            }
            return text;
        }

        async function sha256(plain) {
            const encoder = new TextEncoder();
            const data = encoder.encode(plain);
            const digest = await crypto.subtle.digest('SHA-256', data);
            return base64UrlEncode(digest);
        }

        function base64UrlEncode(buffer) {
            const base64 = btoa(String.fromCharCode(...new Uint8Array(buffer)));
            return base64.replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
        }

        // 存储临时值
        let pkceVerifier = null;
        let pkceState = null;

        // 发起授权请求
        async function startAuth() {
            pkceVerifier = generateRandomString(64);
            const codeChallenge = await sha256(pkceVerifier);
            pkceState = generateRandomString(16);

            // 保存到 sessionStorage 用于回调恢复
            sessionStorage.setItem('pkce_verifier', pkceVerifier);
            sessionStorage.setItem('oauth_state', pkceState);

            const authUrl = `${AUTHORIZE_URL}?` +
                `response_type=code&` +
                `client_id=${encodeURIComponent(CLIENT_ID)}&` +
                `redirect_uri=${encodeURIComponent(REDIRECT_URI)}&` +
                `state=${pkceState}&` +
                `code_challenge=${codeChallenge}&` +
                `code_challenge_method=S256&` +
                `scope=openid profile email`;
            window.location.href = authUrl;
        }

        // 处理回调 (在页面加载时检查 URL 参数)
        async function handleCallback() {
            const urlParams = new URLSearchParams(window.location.search);
            const code = urlParams.get('code');
            const state = urlParams.get('state');
            const error = urlParams.get('error');

            if (error) {
                document.getElementById('result').innerText = `授权失败: ${error}`;
                return;
            }

            if (!code) return;

            const savedVerifier = sessionStorage.getItem('pkce_verifier');
            const savedState = sessionStorage.getItem('oauth_state');

            if (state !== savedState) {
                document.getElementById('result').innerText = '状态参数不匹配，可能遭受 CSRF 攻击';
                return;
            }

            if (!savedVerifier) {
                document.getElementById('result').innerText = '缺少 PKCE verifier';
                return;
            }

            // 换取 token
            try {
                const tokenResponse = await fetch(TOKEN_URL, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
                    body: new URLSearchParams({
                        grant_type: 'authorization_code',
                        client_id: CLIENT_ID,
                        code: code,
                        redirect_uri: REDIRECT_URI,
                        code_verifier: savedVerifier,
                    }),
                });
                const tokenData = await tokenResponse.json();
                if (!tokenResponse.ok) {
                    throw new Error(tokenData.error_description || 'Token exchange failed');
                }
                document.getElementById('result').innerText = JSON.stringify(tokenData, null, 2);
                // 获取用户信息
                const userInfoRes = await fetch(USERINFO_URL, {
                    headers: { 'Authorization': `Bearer ${tokenData.access_token}` },
                });
                const userInfo = await userInfoRes.json();
                document.getElementById('result').innerText += '\n\n用户信息:\n' + JSON.stringify(userInfo, null, 2);
                // 清除存储
                sessionStorage.removeItem('pkce_verifier');
                sessionStorage.removeItem('oauth_state');
                // 可选：去掉 URL 中的 code
                window.history.replaceState({}, document.title, window.location.pathname);
            } catch (err) {
                document.getElementById('result').innerText = `错误: ${err.message}`;
            }
        }

        // 绑定按钮事件
        document.getElementById('loginBtn').addEventListener('click', startAuth);
        // 自动处理回调
        handleCallback();
    </script>
</body>
</html>

将上述代码保存为 .html 文件，替换 CLIENT_ID 和 REDIRECT_URI，用本地 HTTP 服务器打开（不能直接用 file:// 协议，否则 crypto 可能受限），即可测试完整登录流程。

📱 在 React / Vue / Svelte 中使用

你可以将上述逻辑封装成一个自定义 Hook 或服务。推荐使用 @brainoauth/pkce-client（即将发布）或参考以下核心步骤：

  • 在登录按钮点击时调用 startAuth() 生成 verifier 和 challenge 并重定向。
  • 在回调路由组件中，调用 handleCallback() 换取 token 并保存到状态管理或 localStorage（注意安全）。
  • 所有 API 请求时在 Authorization: Bearer <access_token> 头中携带 token。

🌐 移动端 / 桌面端集成

对于 React Native 或 Electron，推荐使用 AppAuth 库（支持 PKCE）。以下为 React Native 示例（使用 react-native-app-auth）： 

import { authorize } from 'react-native-app-auth';

const config = {
  issuer: 'https://your-brain-oauth.com',
  clientId: '你的公共客户端client_id',
  redirectUrl: 'com.yourapp:/callback',
  scopes: ['openid', 'profile', 'email'],
  usePKCE: true,  // 启用 PKCE
};

const result = await authorize(config);
console.log(result.accessToken);

Android / iOS 原生开发可使用 AppAuth SDK，配置方式类似。

🛡️ 安全注意事项

  • 始终使用 HTTPS（生产环境），防止中间人攻击。localhost 开发时可使用 HTTP。
  • 验证 state 参数：防止 CSRF，上述示例已包含。
  • token 存储：access_token 不应存储在 localStorage（XSS 风险），建议保存在内存中（如 React 状态）或带 HttpOnly 的 cookie（需要后端）。对于纯前端应用，可接受短期内存存储。
  • 刷新 token：Brain OAuth 返回 refresh_token 时，建议在后端代理刷新，或使用 PKCE 的刷新流程（需服务端支持）。

❓ 常见问题

Q: 我必须使用 PKCE 吗？

A: 如果你开发的是纯前端应用（无后端服务器），则必须使用 PKCE，否则无法安全保管 client_secret。Brain OAuth 要求公共客户端强制使用 PKCE。

Q: 回调地址支持哪些格式？

A: 支持 HTTPS、HTTP localhost 以及自定义 URL Scheme（用于移动端，如 myapp://callback）。

Q: 如何获取用户信息？

A: 拿到 access_token 后，调用 /userinfo 端点，使用 Bearer Token 认证，返回标准 OIDC 用户声明。

Q: 我的应用是服务端渲染（Next.js 服务端），应该用哪种方式？

A: 服务端应用请使用标准授权码流程（机密客户端），由后端保管 client_secret，更安全。本 PKCE 文档专为纯前端设计。
